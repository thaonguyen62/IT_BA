Tạo & điều chỉnh hóa đơn

01

Mindmap

mindmap
  root((Hóa đơn))
    TẠO HĐ
      Trigger
        In-app: thanh toán xong
        CRM: CS tạo, có csTicketId
      Platform Service
        Lấy trans theo Tid
        Validate
        Tracking = Init
      ERP
        Timeout: Error
        202: Pending
        Callback: Success
      AF
        Email HĐ cho user
    ĐIỀU CHỈNH / HUỶ
      Tính hoàn
        Hotel tính tiền hoàn
        Partner trả tiền charge
      Điều kiện
        Huỷ 1 phần: Partial
        Không huỷ được: ẩn nút
        Huỷ toàn bộ: Full
      Kết quả Success
        Full: HĐ ĐC = 0 + HĐ gốc
        Partial: HĐ ĐC + HĐ gốc
        ERP xuất HĐ thay thế
        AF gửi mail HĐ ĐC
        Push noti user
      Rẽ theo tracking
        Not found: no-op
        Init: cancel request
        Pending: CancelERPRecordTask
        Error/Cancel: no-op

03

Sequence — tạo hóa đơn

sequenceDiagram
    actor User
    participant CRM
    participant Platform Service
    participant ERP
    participant AF Service

    alt In-app payment
        User->>Platform Service: Thanh toán thành công (lưu Pocket)
    else CS tạo trên CRM
        CRM->>Platform Service: Request tạo HĐ (csTicketId)
    end
    Platform Service->>Platform Service: Validate request, lấy trans theo Tid
    Platform Service->>Platform Service: Tạo trackingInvoiceStatus = Init
    Platform Service->>ERP: Gửi request xuất HĐ
    alt Error / Timeout
        ERP-->>Platform Service: Error / Timeout
        Platform Service->>Platform Service: update status = Error/Cancel
        Platform Service-->>User: update status (fail)
    else 202 Accepted
        ERP-->>Platform Service: 202
        Platform Service->>Platform Service: update status = Pending
        ERP->>AF Service: Request gửi mail HĐ
        AF Service-->>User: Email hóa đơn
        ERP-->>Platform Service: Callback success
        Platform Service->>Platform Service: update status = Success, lưu DB
        Platform Service-->>User: update status
    end

Legend trạng thái: Init · Pending · Success · Error/Cancel

04

Sequence — điều chỉnh / huỷ

sequenceDiagram
    actor User
    participant Platform Service
    participant Hotel Service
    participant Partner
    participant ERP
    participant AF Service

    User->>Platform Service: Huỷ booking (1 phòng / toàn bộ) trong Pocket
    Platform Service->>Hotel Service: Call huỷ
    Hotel Service->>Partner: Request huỷ
    Partner-->>Hotel Service: Số tiền bị charge
    Hotel Service->>Hotel Service: Tính tiền hoàn
    Hotel Service->>Platform Service: Lưu thông tin huỷ
    Platform Service->>Platform Service: Query trackingInvoice theo Tid

    alt Không tồn tại / Error-Cancel
        Platform Service->>Platform Service: Không làm gì thêm
    else Init
        Platform Service->>Platform Service: Cancel request
    else Pending
        Platform Service->>ERP: CancelERPRecordTask
        ERP-->>Platform Service: status = CANCEL
        Platform Service->>Platform Service: Chặn xuất hóa đơn
    else Success
        alt Partial — huỷ 1 phần
            Platform Service->>Platform Service: Build HĐ điều chỉnh phần huỷ + giữ HĐ gốc
        else Full — huỷ toàn bộ
            Platform Service->>Platform Service: Build HĐ điều chỉnh = 0 + kèm HĐ gốc
        end
        Platform Service->>ERP: Gửi HĐ điều chỉnh
        ERP->>ERP: Xuất HĐ thay thế
        ERP->>AF Service: Request gửi mail HĐ điều chỉnh
        AF Service-->>User: Email HĐ điều chỉnh
        ERP-->>Platform Service: Response kết quả
    end
    Platform Service-->>User: Push notification
05

Activity — tạo hóa đơn

flowchart TD
    A1([User thanh toán trong app]) --> B1[Hotel Service lưu Pocket]
    A2([CS tạo request trên CRM]) --> B2[CRM generate csTicketId]
    B1 --> C[Platform Service nhận request]
    B2 --> C
    C --> D[Validate request]
    D --> E[Lấy transaction theo Tid]
    E --> F[Tạo trackingInvoiceStatus = Init]
    F --> G[Gửi request xuất HĐ đến ERP]
    G --> H{ERP phản hồi?}
    H -->|Error/Timeout| I[Update status = Error/Cancel]
    I --> Z1([Kết thúc])
    H -->|202 Accepted| J[Update status = Pending]
    J --> K[ERP gửi request đến AF Service]
    K --> L[AF Service gửi mail HĐ cho user]
    L --> M[ERP callback về Platform Service]
    M --> N[Update status = Success, lưu DB]
    N --> O[Update status trên CRM]
    O --> Z2([Kết thúc])
06

Activity — huỷ

flowchart TD
    A([User huỷ booking trong Pocket]) --> B[Platform Service call Hotel Service]
    B --> C[Hotel Service gọi Partner]
    C --> D[Partner trả số tiền charge]
    D --> E[Hotel Service tính tiền hoàn]
    E --> F[Hotel Service lưu info huỷ → Platform]
    F --> G{Query trackingInvoice theo Tid}
    G -->|Không tồn tại / Error/Cancel| Z1([Kết thúc: không làm gì])
    G -->|Init| CR[Cancel request] --> Z2([Kết thúc])
    G -->|Pending| P1[CancelERPRecordTask] --> P2[ERP update = CANCEL] --> P3[Chặn xuất hóa đơn] --> P4[Push noti: đã huỷ, không xuất HĐ] --> Z2
    G -->|Success| S{Loại huỷ}
    S -->|Partial 1 phần| S1[Build HĐ điều chỉnh phần huỷ + giữ HĐ gốc]
    S -->|Full toàn bộ| S2[Build HĐ điều chỉnh = 0 + kèm HĐ gốc]
    S1 --> T[Gửi HĐ điều chỉnh đến ERP]
    S2 --> T
    T --> U[ERP xuất HĐ thay thế]
    U --> V[ERP gọi AF Service]
    V --> W[AF gửi mail HĐ điều chỉnh cho user]
    W --> X[ERP response kết quả về Platform]
    X --> Y[Push notification cho user]

Legend trạng thái: Init · Pending · Success · Error/Cancel
