INVOICE LIFECYCLE — HOTEL BOOKING

Tạo & điều chỉnh hóa đơn

00

Context

  • Expedia: MoMo là forwarder (trung gian) — không trực tiếp quyết định giá bán, nên không phải bên xuất HĐ.
  • OMH: MoMo trực tiếp quyết định giá bán vì nhận giá B2B rồi cộng markup → MoMo là bên bán (principal) → phải xuất HĐ.
  • Song song: luồng xuất HĐ bên Platform (B2C) đã và đang triển khai; phần này chỉ implement việc truyền thông tin HĐ sang cho Platform.

Mindmap

mindmap
  root((Hóa đơn))
    TẠO HĐ
      Trigger
        In-app: thanh toán xong
        CRM: CS tạo, có csTicketId
      Platform Service
        Lấy trans theo Tid
        Validate
        Tracking = Init
      ERP
        Timeout: Error
        202: Pending
        Callback: Success
      AF
        Email HĐ cho user
    ĐIỀU CHỈNH / HUỶ
      Tính hoàn
        Hotel tính tiền hoàn
        Partner trả tiền charge
      Điều kiện
        Huỷ 1 phần: Partial
        Không huỷ được: ẩn nút
        Huỷ toàn bộ: Full
      Kết quả Success
        Full: HĐ ĐC = 0 + HĐ gốc
        Partial: HĐ ĐC + HĐ gốc
        ERP xuất HĐ thay thế
        AF gửi mail HĐ ĐC
        Push noti user
      Rẽ theo tracking
        Not found: no-op
        Init: cancel request
        Pending: CancelERPRecordTask
        Error/Cancel: no-op

Sequence — tạo hóa đơn

sequenceDiagram
    actor User as Khách hàng
    actor CS as Nhân viên CS
    participant Hotel as Hotel Service
    participant CRM
    participant Platform as Platform Service
    participant ERP
    participant VACOM
    participant AF as AF Service

    User->>Hotel: Đặt phòng, mở màn Pax Info

    alt Tick Xuất HĐ và điền info
        User->>Hotel: Nhập info HĐ
        Hotel->>Hotel: Validate FE
        Hotel->>Platform: Lưu info HĐ theo booking
    else Không tick
        User->>CS: Liên hệ yêu cầu xuất HĐ
        CS->>CRM: Nhập info HĐ của khách
        CRM->>Platform: Tạo request HĐ
    end

    activate Platform
    Platform->>Platform: Validate BE

    alt Không hợp lệ
        Platform->>Platform: Status Error
        Platform-->>CRM: Sync trạng thái Error
        Platform-->>User: Báo lỗi
    else Hợp lệ
        Platform->>Platform: Tạo trackingInvoiceStatus Init
        Platform->>ERP: Gửi request xuất HĐ
        activate ERP

        ERP-->>Platform: Accepted
        Platform->>Platform: Status Pending
        Platform-->>CRM: Sync trạng thái Pending

        Note over ERP: Batch xử lý theo lịch

        ERP->>ERP: Tạo danh sách HĐ cần xuất
        ERP->>VACOM: Call API xuất HĐ VAT
        activate VACOM

        alt VACOM lỗi hoặc Timeout
            VACOM-->>ERP: Fail
            ERP-->>Platform: Status Error
            Platform-->>CRM: Sync trạng thái Error
            Platform-->>User: Thông báo thất bại
        else Xuất thành công
            VACOM-->>ERP: Xuất HĐ thành công
            ERP->>AF: Request gửi mail HĐ VAT
            AF-->>User: Email hóa đơn VAT
            ERP-->>Platform: Callback success
            Platform->>Platform: Status Success
            Platform-->>CRM: Sync trạng thái Success
            Platform-->>User: Push notification thành công
        end
        deactivate VACOM

        deactivate ERP
    end

    deactivate Platform
04

Sequence — điều chỉnh / huỷ

sequenceDiagram
    actor User as Khách hàng
    participant CRM
    participant Platform Service
    participant Hotel Service
    participant Partner
    participant ERP
    participant VACOM
    participant AF Service

    User->>+Platform Service: Mở booking trong Pocket
    Platform Service->>Platform Service: Kiểm tra booking có huỷ được không
    alt Không huỷ được
        Platform Service-)User: Ẩn nút "Huỷ" tại màn Pocket
    else Huỷ được
        Platform Service-)User: Hiển thị nút "Huỷ"
        User->>Platform Service: Bấm Huỷ booking (1 phòng / toàn bộ)
        Platform Service->>+Hotel Service: Call huỷ
    Hotel Service->>+Partner: Request huỷ
    Partner-->>-Hotel Service: Số tiền bị charge
    Hotel Service->>Hotel Service: Tính tiền hoàn
    Hotel Service-->>-Platform Service: Lưu thông tin huỷ
    Platform Service->>Platform Service: Query trackingInvoice theo Tid

    alt Không tồn tại / Error-Cancel
        Platform Service->>Platform Service: Không làm gì thêm
    else Init
        Platform Service->>Platform Service: Cancel request
        Platform Service-)CRM: Sync trạng thái = Cancel
    else Pending
        Platform Service->>+ERP: CancelERPRecordTask
        ERP-->>-Platform Service: status = CANCEL
        Platform Service->>Platform Service: Chặn xuất hóa đơn
        Platform Service-)CRM: Sync trạng thái = Cancel
    else Success
        alt Partial — huỷ 1 phần
            Platform Service->>Platform Service: Build HĐ điều chỉnh phần huỷ + giữ HĐ gốc
        else Full — huỷ toàn bộ
            Platform Service->>Platform Service: Build HĐ điều chỉnh = 0 + kèm HĐ gốc
        end
        Platform Service->>+ERP: Gửi HĐ điều chỉnh (vào hàng chờ)
        ERP-->>Platform Service: 202 Accepted (đã nhận)
        Platform Service->>Platform Service: status = Pending
        Platform Service-)CRM: Sync trạng thái = Pending
        Note over ERP: Cron/timer theo lịch (batch) tại ERP — không realtime
        ERP->>ERP: Tạo danh sách HĐ điều chỉnh cần xuất
        ERP->>+VACOM: Call API xuất HĐ thay thế
        VACOM-->>-ERP: Xuất HĐ thay thế thành công
        ERP-)AF Service: Request gửi mail HĐ điều chỉnh
        AF Service-)User: Email HĐ điều chỉnh
        ERP-)Platform Service: Callback success
        Platform Service->>Platform Service: status = Success (đã điều chỉnh)
        Platform Service-)CRM: Sync trạng thái = Success
        deactivate ERP
    end
        Platform Service-)User: Push notification
    end
    deactivate Platform Service
