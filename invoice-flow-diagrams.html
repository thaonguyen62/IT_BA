Invoice lifecycle — hotel booking

Tạo & điều chỉnh hóa đơn: bộ sơ đồ chốt

Bảy góc nhìn của cùng một luồng nghiệp vụ — từ tổng quan (mindmap, use case) đến chi tiết thực thi (sequence, activity, BPMN lane) — dựa trên bản mô tả đã thống nhất giữa Platform, Hotel, ERP, AF Service.

01

Mindmap

Toàn cảnh hai nhánh nghiệp vụ chính — tạo hóa đơn và điều chỉnh/huỷ — cùng các nhánh con theo service sở hữu.

mindmap
  root((Hóa đơn))
    (TẠO HĐ)
      Trigger
        In-app: thanh toán xong
        CRM: CS tạo, có csTicketId
      Platform Service
        Validate
        Lấy trans theo Tid
        Tracking = Init
      ERP
        Timeout: Error
        202: Pending
        Callback: Success
      AF
        Email HĐ cho user
    (ĐIỀU CHỈNH / HUỶ)
      Điều kiện
        Không huỷ được: ẩn nút
        Huỷ 1 phần: Partial
        Huỷ toàn bộ: Full
      Tính hoàn
        Partner trả tiền charge
        Hotel tính tiền hoàn
      Rẽ theo tracking
        Not found: no-op
        Error/Cancel: no-op
        Init: cancel request
        Pending: CancelERPRecordTask
        Success: build HĐ điều chỉnh
      Kết quả Success
        Partial: HĐ ĐC + HĐ gốc
        Full: HĐ ĐC = 0 + HĐ gốc
        ERP xuất HĐ thay thế
        AF gửi mail HĐ ĐC
        Push noti user

03

Sequence — tạo hóa đơn

Hai nguồn trigger hội tụ về Platform Service; nhánh ERP rẽ theo error/timeout hoặc 202 + callback bất đồng bộ.

sequenceDiagram
  actor U as User
  participant CRM
  participant PF as Platform Service
  participant ERP
  participant AF as AF Service

  alt In-app payment
    U->>PF: Thanh toán thành công (lưu Pocket)
  else CS tạo trên CRM
    CRM->>PF: Request tạo HĐ (csTicketId)
  end
  PF->>PF: Validate request, lấy trans theo Tid
  PF->>PF: Tạo trackingInvoiceStatus = Init
  PF->>ERP: Gửi request xuất HĐ

  alt Error / Timeout
    ERP-->>PF: Error / Timeout
    PF->>PF: update status = Error/Cancel
    PF->>CRM: update status (fail)
  else 202 Accepted
    ERP-->>PF: 202
    PF->>PF: update status = Pending
    ERP->>AF: Request gửi mail HĐ
    AF-->>U: Email hóa đơn
    ERP-->>PF: Callback success
    PF->>PF: update status = Success, lưu DB
    PF->>CRM: update status
  end
Init Pending Success Error/Cancel
04

Sequence — điều chỉnh / huỷ

Hotel Service và Partner tính tiền hoàn trước; Platform Service rẽ nhánh theo trạng thái tracking hiện tại của HĐ gốc.

sequenceDiagram
  actor U as User
  participant PF as Platform Service
  participant HTL as Hotel Service
  participant PT as Partner
  participant ERP
  participant AF as AF Service

  U->>PF: Huỷ booking (1 phòng / toàn bộ) trong Pocket
  PF->>HTL: Call huỷ
  HTL->>PT: Request huỷ
  PT-->>HTL: Số tiền bị charge
  HTL->>HTL: Tính tiền hoàn
  HTL->>PF: Lưu thông tin huỷ
  PF->>PF: Query trackingInvoice theo Tid

  alt Không tồn tại / Error-Cancel
    PF->>PF: Không làm gì thêm
  else Init
    PF->>PF: Cancel request
  else Pending
    PF->>ERP: CancelERPRecordTask
    ERP-->>PF: status = CANCEL
    PF->>PF: Chặn xuất hóa đơn
  else Success
    alt Partial — huỷ 1 phần
      PF->>PF: Build HĐ điều chỉnh phần huỷ + giữ HĐ gốc
    else Full — huỷ toàn bộ
      PF->>PF: Build HĐ điều chỉnh = 0 + kèm HĐ gốc
    end
    PF->>ERP: Gửi HĐ điều chỉnh
    ERP->>ERP: Xuất HĐ thay thế
    ERP->>AF: Request gửi mail HĐ điều chỉnh
    AF-->>U: Email HĐ điều chỉnh
    ERP-->>PF: Response kết quả
    PF-->>U: Push notification
  end
05

Activity — tạo hóa đơn

Hai điểm khởi tạo (in-app / CS-CRM) hội tụ về Platform Service, rẽ nhánh theo phản hồi ERP.

flowchart TD
  A1(["User thanh toán trong app"]) --> C1["Hotel Service lưu Pocket"]
  A2(["CS tạo request trên CRM"]) --> C2["CRM generate csTicketId"]
  C1 --> D["Platform Service nhận request"]
  C2 --> D
  D --> E["Validate request"]
  E --> F["Lấy transaction theo Tid"]
  F --> G["Tạo trackingInvoiceStatus = Init"]
  G --> H["Gửi request xuất HĐ đến ERP"]
  H --> I{"ERP phản hồi?"}
  I -- "Error / Timeout" --> J["Update status = Error/Cancel"] --> Z1(["Kết thúc"])
  I -- "202 Accepted" --> K["Update status = Pending"]
  K --> L["ERP gửi request đến AF Service"]
  L --> M["AF Service gửi mail HĐ cho user"]
  M --> N["ERP callback về Platform Service"]
  N --> O["Update status = Success, lưu DB"]
  O --> P["Update status trên CRM"]
  P --> Z2(["Kết thúc"])
Init Pending Success Error/Cancel
06

Activity — huỷ

Toàn bộ nhánh quyết định của luồng huỷ, từ tính tiền hoàn đến từng kết cục theo trạng thái tracking.

flowchart TD
  A(["User huỷ booking trong Pocket"]) --> A2["Platform Service call Hotel Service"]
  A2 --> B["Hotel Service gọi Partner"]
  B --> C["Partner trả số tiền charge"]
  C --> D["Hotel Service tính tiền hoàn"]
  D --> E["Hotel Service lưu info huỷ → Platform"]
  E --> F{"Query trackingInvoice theo Tid"}
  F -- "Không tồn tại" --> Z1(["Kết thúc: không làm gì"])
  F -- "Error/Cancel" --> Z1
  F -- "Init" --> G["Cancel request"] --> Z2(["Kết thúc"])
  F -- "Pending" --> H["CancelERPRecordTask"] --> I["ERP update = CANCEL"] --> J["Chặn xuất hóa đơn"]
  J --> Jn["Push noti: đã huỷ, không xuất HĐ"] --> Z2
  F -- "Success" --> K{"Loại huỷ"}
  K -- "Partial (1 phần)" --> L["Build HĐ điều chỉnh phần huỷ + giữ HĐ gốc"]
  K -- "Full (toàn bộ)" --> M["Build HĐ điều chỉnh = 0 + kèm HĐ gốc"]
  L --> N["Gửi HĐ điều chỉnh đến ERP"]
  M --> N
  N --> O["ERP xuất HĐ thay thế"]
  O --> Oaf["ERP gọi AF Service"]
  Oaf --> Omail["AF gửi mail HĐ điều chỉnh cho user"]
  Omail --> P["ERP response kết quả về Platform"]
  P --> Q(["Push notification cho user"])
07

BPMN (dạng lane)

Luồng tạo hóa đơn theo pool/lane từng service — thể hiện điểm bàn giao trách nhiệm và event bất đồng bộ (202 → callback).

flowchart TB
  subgraph LaneUser["Lane: User / CRM"]
    S1(("Start: thanh toán / tạo request"))
  end
  subgraph LanePF["Lane: Platform Service"]
    P1["Validate + lấy trans theo Tid"]
    P2["Tạo tracking = Init"]
    P3{"ERP response?"}
    P4["Update = Error/Cancel"]
    P5["Update = Pending"]
    P6["Update = Success + lưu DB + update CRM"]
  end
  subgraph LaneERP["Lane: ERP Service"]
    E1["Xử lý xuất HĐ"]
    E2["Callback success"]
  end
  subgraph LaneAF["Lane: AF Service"]
    A1["Gửi mail HĐ"]
  end

  S1 --> P1 --> P2 --> E1
  E1 -- "response" --> P3
  P3 -- "Error/Timeout" --> P4 --> END1(("End"))
  P3 -- "202" --> P5
  E1 -. "sau 202: trigger mail" .-> A1
  A1 --> E2
  E2 -- "callback" --> P6 --> END2(("End"))
Bộ sơ đồ dựng từ bản mô tả nghiệp vụ đã chốt — Platform / Hotel / ERP / AF Service, trạng thái tracking Init → Pending → Success/Error.
